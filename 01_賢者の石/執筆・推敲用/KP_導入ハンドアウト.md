# KP用・導入配役（継続探索者推奨）

**対象**: 第1話「黄昏の失楽園と賢者の石」  
**用途**: セッション0〜各PCの導入を進行する**KP用資料**。`KP_導入1〜5` と合わせて使う。  
**PCには配布しない**（口頭ロールプレイで伝える。調査文案は `PL_*` を別途）。  
**正本**: `導入_整理とハンドアウト設計.md`

---

## このシナリオの前提

| 項目 | 方針 |
|------|------|
| **探索者** | **継続探索者推奨**。すでに神話的事象に遭遇した経験があるPCを使う |
| **新規PC** | 可。下記「新規向けの一言」を足す |
| **職業縛り** | **なし**。導入は「職業」ではなく**状況**で割り当てる |
| **人数** | 基本4人。3人は1状況省略、5人は状況を1つ追加 |

**意図**: プレイヤーが持ち込んだ探索者のバックグラウンド・人脈・技能に合わせて入口を選び、無理な職業変更や新規作成を避ける。

※ BadAppleの店を知っているか・見つけ方 → `KP_人物_黒須.md`「探索者とBadApple」（導入資料では扱わない）

---

## KPの作業手順（セッション0）

### 1. 探索者リストを確認する

各PCについて、次をメモする（プレイヤーに口頭で聞く・キャラシートで確認）。

| 確認項目 | 導入割当てに効く |
|----------|------------------|
| 赤霧市／御津門大学との縁 | 学生・教職員・地元か |
| **黄昏の失楽園**との関係 | 未接触／噂のみ／仲間あり |
| **御手神舞・安心院** | 面識の有無 |
| 得意技能 | コンピューター・図書館・信用など |
| 表の仕事・バックグラウンド | B〜Dの読み替えに使う |

### 2. 状況を割り当てる（下「状況カタログ」）

- **4人**: ルートA〜Dから**各1つ**（重複可だが、情報の偏りに注意）
- **3人**: **ルートCを省略推奨**（噂の**存在**は下記「オカルト等技能」で補填。5ch・隠しHPの**詳細**は合流後の調査）
- **5人**: ルートEを**1つ追加**

### 3. KPが使う資料と、PLに渡すものの区別

| 種別 | ファイル | 使い方 |
|------|----------|--------|
| **KP用・導入** | `KP_導入1〜5_*.md` | 配役したルートの進行台本。**配布しない** |
| **KP用・配役** | 本ファイル | 状況選択・マッチング |
| **PL用**（調査後） | `PL_噂調査_賢者の石.md` `PL_5chスレッド.md` `PL_被害者リスト.md` 等 | 調査成功・場面到達時に渡す |
| **PL用**（任意） | `PL_黄昏の失楽園_紹介.md` | 失楽園未接触のPCに、組織の噂として |

### 4. 合流を宣言する

**目安**: 5/24(月) **20時以降・BadApple**  
**導入1**のPCがスマホを持つ。他PCの来店理由は各導入＋`KP_人物_黒須.md`参照。

## 5人・ペア導入（2人で同じ導入）

5人目（ルートE）を、A〜Dのいずれかのシーンに**居合わせる**形で進行する。**導入5単独は次善策**。

### 選び方

| 手順 | 内容 |
|------|------|
| 1 | A〜Dを4人に割り当て（4人構成と同じ） |
| 2 | 5人目に **導入5（ルートE）** を割り当て |
| 3 | EのPCが**他ルートのシーンに自然に入れるか**を見る |
| 4 | 入れるルートを1つ選び、**同じ時刻・同じ場面**で2人同時にロールプレイ |

### ペア候補（優先順）

| ペア | 向く組み合わせ | 5人目の立場 | 主役PCが持つ情報 |
|------|----------------|-------------|------------------|
| **E＋A** ◎ | 帰路・一松方面の探索者 ＋ 大学帰りの学生 | **現場の目撃者**（路地で居合わせる） | **A**：スマホ・BadApple遺言 |
| **E＋D** ○ | 大学生（D-2）＋ オカ研部員 | 部室・キャンパスで**同席** | **D**：舞の顔見せ・依頼・金田 |
| **E＋C** ○ | ネット／オカルト ＋ オカ研部員 | 5chの **KND★＝先輩** と気づく | **C**：噂・5ch・隠しHP入口 |
| **E＋B** △ | 聞き込み・取材 ＋ 学生 | 一松で**偶然遭遇**・一言添える | **B**：被害1〜3のリスト |
| **E単独** | 上記がどれも不自然 | オカ研部室ソロ（`KP_導入5` 標準） | Eのみ：部内の空気 |

※ **E＋A** が最も効く。ユーザー例「Aの現場に居合わせる」はこのペア。

### 情報の分け方（ペア導入共通）

| 原則 | 内容 |
|------|------|
| **必須成果は主役ルートのPCが持つ** | 例：スマホはA、被害リストはB、5ch到達はC、舞依頼はD |
| **Eは文脈・感情・補助技能** | 金田の顔・LINE・オカ研の噂。技能成功で主役の発見を後押ししてよい |
| **重複OK** | 2人とも「変死は血痕がない」を見てよい。SANは各自 |
| **合流** | ペアの2人は最初から**同行前提**でBadAppleへ向かいやすい |

### ペア別・入口の一言（KP用）

**E＋A** — `KP_導入1` ＋ `KP_導入5`「E＋A」
